Fallback
Fallback Details

If you look at windows volume mixer, you will see that not all applications can have their volume controlled. The fallback behavior controls what happens when you are in an application that doesn't show up in the volume mixer

* System Sounds - Switch to system sounds. This will control windows sound effects such as when an error sound plays. If you're in an application that is making beeping sounds, this will often allow you to control those sounds while leaving things like your music/videos alone

* Previous App - Use the last app that had a volume control. This can result in the stream deck not changing after you have quit an application.

* Default Output Device Volume - Switch to the main volume control for the default output device. This will change the volume of the default output device. This is usually volume for all applications, unless you override the output device for specific applications.

Overrides
Override Details

Some games use anti-cheat software that interferes with the ability to know what application is running, and pair it to the appropriate audio process.

There is nothing I can do about this.

In order to work around this, the overrides mechanism is there for you to manually map an application window to an audio process.

They synax for this is

[matching]: Window Title
Audio Process

Blank lines can be used for spacing
Lines starting with // are ignored

[matching] can be eq, start, end, or regex. Which will perform an exact match, starts with, ends with, or regular expressions respectively.

Example:
eq: Task Manager
